Sport Link — Compilar APK com Expo/EAS

Passo a passo completo para gerar o APK Android do Sport Link usando EAS Build no Windows.

Navegação

1. Visão geral

O APK do Sport Link é gerado pelo EAS Build. O projeto é enviado para os servidores da Expo, entra em uma fila, é compilado e no final a Expo libera um link para baixar o arquivo .apk.

📁

Projeto local

Você mantém os arquivos do app no computador.

☁️

EAS Build

O EAS envia e compila o projeto nos servidores da Expo.

📲

Download APK

No final, a Expo mostra um link para baixar o APK.

O APK não aparece automaticamente dentro da pasta do projeto. O APK fica disponível para download na página do build da Expo.

4. app.json correto

O app.json define nome do aplicativo, slug, ícone, splash e pacote Android.

app.json
{
  "expo": {
    "name": "Sport Link",
    "slug": "sport-link",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "userInterfaceStyle": "dark",
    "newArchEnabled": true,
    "splash": {
      "image": "./assets/splash-icon.png",
      "resizeMode": "contain",
      "backgroundColor": "#121212"
    },
    "ios": {
      "supportsTablet": true
    },
    "android": {
      "package": "com.sportlink.app",
      "versionCode": 1,
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#121212"
      },
      "edgeToEdgeEnabled": true
    },
    "web": {
      "favicon": "./assets/favicon.png"
    }
  }
}
O campo android.package é o identificador único do Android: com.sportlink.app.

5. package.json correto

O package.json define dependências e scripts. O script build:apk chama o EAS com o perfil preview.

{
  "name": "sport-link",

    "android": "expo start --android",

6. eas.json correto

O eas.json define o tipo de build. Para instalar no celular, o perfil preview deve gerar apk.

eas.json
{
  "cli": {
    "version": ">= 5.7.0"
  },
  "build": {
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}
Não use "aab" em buildType. O valor correto é "app-bundle".

8. Erro do Git e EAS_NO_VCS

Se aparecer erro dizendo que o Git não foi encontrado, use o modo sem controle de versão.

Ativar modo sem Git
set EAS_NO_VCS=1

Depois rode o build normalmente:

Build sem Git
set EAS_NO_VCS=1
npx eas-cli@latest build -p android --profile preview
Esse comando resolveu o erro de Git no seu computador.

9. Gerar o APK

Com tudo pronto, rode os comandos abaixo dentro da pasta correta do projeto.

Comandos recomendados
npm install
set EAS_NO_VCS=1
npx eas-cli@latest login
npx eas-cli@latest build -p android --profile preview

Se já estiver logado, pode rodar apenas:

Build direto
set EAS_NO_VCS=1
npx eas-cli@latest build -p android --profile preview

10. Entendendo a fila do Expo

Depois que o projeto é enviado, a tela pode mostrar Status: Queued e Free Tier Queue. Isso é normal.

Queued

O build está aguardando um servidor livre.

In progress

O servidor da Expo começou a compilar o app.

Finished

O APK está pronto para download.

Failed

O build falhou e será necessário olhar os logs.

11. Baixar o APK

Quando o build terminar, abra a página do build no Expo e procure o botão de download.

  1. Acesse o link mostrado no terminal.
  2. Espere o status ficar como Finished ou Completed.
  3. Clique em Download, Download APK ou Install.
  4. O arquivo normalmente vai para C:\Users\Guilherme_2\Downloads.
  5. Envie o APK para o celular e instale.
No Android, talvez seja necessário permitir instalação de apps desconhecidos.

12. Erros comuns e soluções

Run this command inside a project directory

Você está em uma pasta acima. Entre na pasta onde existem package.json, app.json e eas.json.

eas não é reconhecido

Use npx eas-cli@latest ou feche e abra o CMD depois da instalação.

buildType must be one of apk, app-bundle

Troque aab por app-bundle no eas.json.

git command not found

Rode set EAS_NO_VCS=1 antes do build.

Arquivo de som não encontrado

Confira se existem assets/sounds/chute.mp3, gol.mp3, trave.mp3, torcida.mp3 e apito.mp3.

Ícone ou splash não encontrado

Confira se existem assets/icon.png, adaptive-icon.png, splash-icon.png e favicon.png.
