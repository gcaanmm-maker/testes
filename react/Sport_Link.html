Visão geral do app

O Sport Link é um app React Native com Expo dividido em módulos. A navegação fica no App.js, as telas ficam em arquivos separados e as configurações compartilhadas ficam em supabase.js e styles.js.

Ideia principal: separar responsabilidades. Assim, Notícias cuida das APIs de conteúdo, Bolão cuida de grupos e palpites, Jogar cuida do mapa/peladas e Joguinho cuida do minigame, áudio, IA e online.

Fluxos principais do app explicados

Aqui estão os caminhos mais importantes do aplicativo explicados como sequência de eventos.

Fluxo da navegação

  1. O Expo carrega o App.js.
  2. App.js cria o NavigationContainer.
  3. Tab.Navigator monta as quatro guias.
  4. Cada Tab.Screen aponta para um arquivo de tela separado.
  5. Bolão e Jogar recebem usuarioLogado por props.

Fluxo das notícias

  1. A tela abre e executa useEffect.
  2. fetchNoticias busca artigos na NewsAPI.
  3. fetchDadosFutebol busca classificação e jogos na Football-Data.
  4. Os estados noticias, jogos e classificacao são atualizados.
  5. FlatList redesenha a tela com os dados novos.
  6. Ao tocar em uma notícia, WebView abre o link dentro do app.

Fluxo do Bolão

  1. Usuário cadastra ou faz login.
  2. App salva o usuário em usuarioLogado.
  3. Usuário cria grupo ou entra com código.
  4. O app salva vínculo em grupo_membros.
  5. Jogos são carregados pela Football-Data API.
  6. Usuário digita placares e salva em bolao.
  7. Ranking lista os membros do grupo ativo.

Fluxo da aba Jogar

  1. Tela carrega partidas do Supabase.
  2. Junta partidas reais com peladas mockadas.
  3. MapView renderiza marcadores usando latitude/longitude.
  4. Usuário preenche formulário de nova pelada.
  5. App valida campos e usuário logado.
  6. Nova partida é inserida na tabela partidas.

Fluxo do Joguinho Online

  1. Jogador cria sala e recebe código.
  2. Sala é salva na tabela jogo_online_salas.
  3. Host abre jogo como Verde.
  4. Convidado entra com código e joga como Vermelho.
  5. WebView envia estado para o React Native.
  6. React Native atualiza o estado no Supabase.
  7. Polling busca o estado e injeta novamente no WebView.

Fluxo do áudio do Joguinho

  1. Expo AV carrega os arquivos mp3.
  2. HTML Canvas detecta chute, gol ou trave.
  3. WebView envia mensagem para React Native.
  4. handleWebViewMessage chama playNativeSound.
  5. Ao sair da guia, useFocusEffect chama stopAllNativeSounds.
  6. A torcida para e não toca nas outras páginas.

SQL completo das tabelas principais

Use estes comandos no Supabase SQL Editor para montar a base do app.

create table if not exists public.usuarios (
  id uuid primary key default gen_random_uuid(),
  email text unique not null,
  senha text not null,
  criado_em timestamptz not null default now()
);

create table if not exists public.grupos (
  id uuid primary key default gen_random_uuid(),
  nome text not null,
  codigo_acesso text unique not null,
  tipo text default 'bolao',
  criador_id uuid,
  criado_em timestamptz not null default now()
);

create table if not exists public.grupo_membros (
  id uuid primary key default gen_random_uuid(),
  grupo_id uuid not null,
  usuario_id uuid not null,
  status text not null default 'aprovado',
  criado_em timestamptz not null default now()
);

create table if not exists public.bolao (
  id text primary key,
  usuario_id uuid not null,
  grupo_id uuid not null,
  jogo text not null,
  placar_time1 text not null,
  placar_time2 text not null,
  criado_em timestamptz not null default now()
);

create table if not exists public.partidas (
  id uuid primary key default gen_random_uuid(),
  nome_quadra text not null,
  endereco text not null,
  data_hora text not null,
  jogadores int,
  goleiros int,
  valor text,
